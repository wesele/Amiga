🏭 内容生产

题库内容通过 Web 端的内容生产系统批量生成。系统调用 OpenAI 兼容的大模型 API，依据题型定义和主题规格自动产出结构化题目数据，经多轮校验与人工审核后入库。本模块定义了完整的 Prompt 工程、输出格式规范、质量控制流程及 Web 界面设计。

🔄 生产流水线

一次完整的题目生产经历五个阶段：

阶段操作产出
① 配置选择目标语言、CEFR 级别、单元主题、题型池、数量配比生产任务描述文件
② 生成按题型 Prompt 模板调用 LLM，批量产出题目 JSON原始题目集（含冗余）
③ 校验自动校验：格式合规、答案正确、难度匹配、查重去重清洗后题目集
④ 审核人工逐条审核：语言自然度、文化适当性、题目可玩性最终题目集
⑤ 入库写入题库数据库，关联单元、级别、主题标签可用题库
✅ 设计原则：LLM 负责草稿生成，人类负责质量把关。AI 产出 3~5 倍冗余量，经自动+人工双重过滤后保留最终题库，确保题目质量不低于手写水平。

🧠 Prompt 工程

每种题型的 Prompt 由角色设定、题型规格、主题约束、输出格式、示例五部分组成。题型规格直接复用「内容结构」中的题型定义，确保 LLM 输出与设计意图一致。

Prompt 结构模板

角色设定：你是一位专业的 {目标语言} 教材编写者，擅长为 {CEFR级别} 学习者设计练习题。

题型规格：请生成 {题型ID}（{题型名称}）类型的题目。交互形式：{交互形式}。训练技能：{训练技能}。认知层次：{认知层次}。

主题约束：主题为"{单元主题}"。词汇范围：{词汇列表}。语法范围：{语法点}。难度：{难度}。

输出格式：严格按照以下 JSON Schema 输出，不要输出其他内容。
{JSON Schema}

示例：{2-3 个该题型的已有人工题目}

题型 Prompt 要点

各题型在通用结构之上，有各自的关键约束：

题型特有约束常见问题与规避
T01 图片识词图片用 emoji 或简短文字描述；干扰项须为同类词；正确答案不能靠排除法得出干扰项过于明显；正确选项与其他项语义重叠
T02 听音选图需标注发音文本（TTS 播放内容）；图片描述须简洁直观；选项间不可有包含关系发音文本含拼写错误；选项图过于抽象
T03 双向配对固定 4 组配对；左右两列各自不重复；配对项须在同一语义域出现多解配对（一词多义）；左右列有相同词
T05 补全句子空位只能是一个词或短语；3~4 个选项含且仅含 1 个正确答案；干扰项须为合理语法变体空位信息不足需语境猜测；干扰项语法明显不通
T06 句子排序拆分单词须完整无遗漏；目标句须语法正确且语义自然；打乱顺序不可与原序相同单词拆分遗漏（如省略标点）；存在多种合法排序
T07 翻译选择源语句子须自然地道；干扰项与正确答案语义相关但错误；不可使用逐字翻译作为正确答案干扰项明显荒谬；正确答案可用排除法秒选
T08 听力选择需标注完整听力文本；选项为文字描述；答案须仅从听力内容即可得出，不需额外知识选项需视觉信息才能判断；听力文本语法错误
T09 拼写输入需标注发音文本和释义；答案仅接受一种拼写（含重音符号）；须注明常见错误拼写答案存在多种合法拼写未标注；缺少重音提示
T10 翻译输入须列出 2~3 种可接受答案；答案须语法正确且自然；避免需要文化背景知识的翻译仅提供一种生硬答案；语法正确但不自然的表达
T11 语音跟读句子长度 5~15 词；须标注评分维度（发音/流利度/完整度）；句子须为日常对话常用句句子过长；包含罕见词汇；书面语过重
T12 情景回应场景描述须简洁清晰；3 个选项中 1 正确 2 干扰；干扰项须在语境中看似合理但有语用缺陷干扰项完全不合理；正确选项过于明显

📐 输出格式规范

所有 LLM 产出的题目必须符合统一的 JSON 格式，便于后续校验、存储和消费。不同题型在通用字段之外，有各自的扩展字段。

通用字段

字段类型说明示例
idstring全局唯一 ID"es-a1-u1-t01-001"
typestring题型 ID"T01"
typeNamestring题型中文名"图片识词"
languagestring目标语言 ISO 639-1"es"
cefrstringCEFR 级别"A1"
unitstring所属单元"U1"
unitThemestring单元主题"Saludos y cortesía"
difficultyinteger难度 1~51
tagsstring[]主题/语法标签["问候", "日常用语"]

题型扩展字段

题型扩展字段说明
T01 图片识词imageDesc, options[], answerIdximageDesc: 图片 emoji 或描述；options: 4 个文字选项；answerIdx: 正确选项索引
T02 听音选图audioText, imageOptions[], answerIdxaudioText: TTS 播放文本；imageOptions: 4 个图片描述；answerIdx: 正确索引
T03 双向配对pairs[{left, right}]pairs: 4 组配对，left 为目标语，right 为源语释义
T05 补全句子sentence, blank, options[], answerIdxsentence: 含 ___ 的完整句；blank: 正确答案；options: 3~4 选项
T06 句子排序words[], targetSentencewords: 打乱后的单词数组；targetSentence: 目标正确句子
T07 翻译选择sourceText, sourceLang, options[], answerIdxsourceText: 源语言句子；options: 4 个目标语言翻译；answerIdx: 正确索引
T08 听力选择audioText, question, options[], answerIdxaudioText: 听力文本；question: 提问；options: 3~4 个文字选项
T09 拼写输入audioText, hint, answer, commonMistakes[]audioText: 发音文本；hint: 释义提示；answer: 正确拼写；commonMistakes: 常见错误
T10 翻译输入sourceText, sourceLang, acceptedAnswers[], hintacceptedAnswers: 2~3 种可接受答案；hint: 可选提示
T11 语音跟读audioText, scoringDimensions[], difficultyNotesscoringDimensions: 评分维度及权重；difficultyNotes: 发音难点提示
T12 情景回应scenario, options[], answerIdx, pragmaticsNotescenario: 场景描述；options: 3 个回应选项；pragmaticsNote: 语用学解释

🔍 质量控制

LLM 生成的题目存在幻觉、格式偏移、难度失配等风险，需通过多层防线确保入库题目质量。

自动校验层

检查项说明不通过处理
格式合规JSON 解析成功，所有必填字段存在且类型正确丢弃，记录日志
答案校验answerIdx 在 options 范围内；blank 与 options[answerIdx] 一致（T05）；targetSentence 语法正确（T06）自动修正或丢弃
语言一致性目标语言内容不含他语混入；源语言翻译准确（可二次调用 LLM 验证）丢弃
难度匹配词汇和语法不超过目标 CEFR 级别（基于 CEFR 词汇表校验）降级或丢弃
查重去重与已有题库做语义相似度比对，重复阈值 > 0.85 视为重复丢弃
选项有效性选择类题型：4 个选项互不重复；配对类：无多解配对丢弃

AI 二次审查

通过自动校验的题目，再用另一个 LLM 调用（或同模型不同 Prompt）进行交叉审查：

审查维度Prompt 要点输出
答案正确性这道题的正确答案是否确实正确？干扰项是否确实错误？pass / fail + 原因
题目可玩性正确答案是否过于明显或可用排除法秒选？pass / warn / fail
文化适当性题目内容是否存在文化冒犯或不适当内容？pass / fail

人工审核

最终由母语者或专业教师逐题审核。审核界面展示题目预览、自动校验结果、AI 审查建议，审核者可一键通过、修改或驳回。

⚠️ 人工审核不可省略。LLM 可能在语法细节（如重音符号、阴阳性一致）上出错，这类错误自动校验难以捕捉，必须由人类把关。

⚙️ 生产任务配置

一次生产任务通过配置文件描述，包含目标语言、级别、单元、题型配比等信息。系统据此自动组装 Prompt 并批量调用 LLM。

配置文件示例

A1 U1 生产任务配置 
{
  "taskId": "es-a1-u1",
  "language": "es",
  "languageName": "西班牙语",
  "cefr": "A1",
  "unit": "U1",
  "unitTheme": "Saludos y cortesía",
  "unitThemeCN": "问候与礼貌",
  "vocabulary": ["hola", "adiós", "buenos días", "buenas tardes",
                 "buenas noches", "gracias", "por favor", "perdón",
                 "lo siento", "de nada", "hasta luego", "hasta mañana",
                 "bien", "mal", "así así"],
  "grammar": ["基本问候句型", "正式/非正式用语区别"],
  "questionTypes": [
    { "type": "T01", "count": 5, "difficulty": [1, 1, 1, 1, 2] },
    { "type": "T02", "count": 5, "difficulty": [1, 1, 1, 1, 2] },
    { "type": "T03", "count": 9, "difficulty": [1,1,1,1,1,1,1,1,2] }
  ],
  "overGenerate": 3,
  "model": "gpt-4o-mini",
  "reviewModel": "gpt-4o"
}

配置字段说明

字段说明
taskId任务唯一标识，格式 {语言}-{级别}-{单元}
vocabulary本单元目标词汇列表，LLM 须在此范围内出题
grammar本单元语法点列表，LLM 须覆盖这些语法
questionTypes题型及数量配比，含每题难度分配
overGenerate过采样倍数，生成 count × N 条以覆盖校验损耗
model主生产模型（成本优先）
reviewModel审查模型（质量优先）
💡 模型选择策略：主生产使用低成本模型（如 gpt-4o-mini）批量生成，审查使用高精度模型（如 gpt-4o）交叉验证。两种角色可使用不同供应商的 OpenAI 兼容 API，系统仅依赖 /chat/completions 接口，与具体模型解耦。

🖥️ Web 界面设计

内容生产系统以 Web 应用的形式提供，面向内容运营人员和审核教师，无需技术背景即可操作。

核心页面

页面功能关键交互
仪表盘总览生产状态、题库统计按语言/级别/单元的题目数量统计；近期生产任务状态；待审核队列数量
任务配置创建和编辑生产任务表单选择语言、级别、单元、题型配比；预览 Prompt；一键启动生产
生产监控实时查看生成进度进度条；每条题目生成状态（等待/生成中/校验中/待审核）；失败重试
审核工作台逐题审核和编辑题目卡片预览（模拟练习界面）；通过/驳回/编辑操作；自动校验结果标注；AI 审查建议悬浮提示
题库管理浏览、搜索、导出题库按语言/级别/单元/题型筛选；批量导出 JSON；题目详情查看与编辑

审核工作台交互流程
