# Soul Mate / 灵伴：故事陪伴学习实现框架

## 1. 产品定义

`Soul Mate`（中文名：`灵伴`）是学习模块内的故事陪伴功能。用户创建一位虚拟伴侣，伴侣每天讲一段连续故事；用户看完后用目标语言与其对话，当天对话、选择和学习表现会进入长期记忆，影响次日剧情，最终走向用户选择的关系目标。

核心循环：

`今日钩子 → 故事阅读/听力 → 伴侣对话 → 关系与剧情结算 → 悬念 → 次日续写`

首版灵伴目标固定为三类：

| 类型 | 核心体验 | 常用剧情 | 关系结局 |
|---|---|---|---|
| 灵魂 | 被理解、信任、深度连接 | 秘密、选择、脆弱时刻、价值观冲突 | 彼此确认、共同生活、长期约定 |
| 解压 | 放松、治愈、幽默陪伴 | 日常糗事、反差笑点、温柔照顾、小确幸 | 稳定陪伴、共同仪式、情绪避风港 |
| 探索 | 好奇、冒险、文化发现 | 旅行、谜题、城市传说、突发事件 | 长期旅伴、共同目的地、下一场冒险 |

## 2. 每日内容规则

每天必须包含以下内容中的至少两项，并避免连续多日使用同一种组合：

- **刺激**：紧迫任务、误会、危险边缘、谜题或关系冲突。
- **暧昧张力**：心动、靠近、吃醋、试探、双关或约会意外。
- **惊喜**：人物反转、旧伏笔回收、意外礼物或场景突变。
- **冷知识**：当地文化、历史、语言习惯、食物、科学或心理知识。

建议配比：每集一个主钩子、一个辅钩子、一个悬念；冷知识必须服务于情节，不单独插播百科。

“擦边”定义为成年人可选的浪漫暧昧与感官张力，不生成露骨性行为、色情描写或胁迫内容。开启成熟暧昧模式前确认用户已满 18 岁，并允许用户随时降低强度。

内容参数：

```text
intensity: 0..3       刺激度
romance_tension: 0..3 暧昧度
surprise: 0..3        反转度
knowledge: 0..3       冷知识密度
comfort: 0..3         治愈度
```

三种灵伴使用不同默认权重，用户可调整；系统根据跳过、继续对话、次日回访等行为缓慢学习偏好。

## 3. 学习页入口布局

学习页现有大布局保持不变：标题、学习统计和“晋级之路”横向进度卡不改。只调整下方小按钮区域为每行 3 个，并加入灵伴：

```text
┌───────────────────────────────┐
│       晋级之路（保持现状）      │
└───────────────────────────────┘
┌──────────┬──────────┬──────────┐
│   新闻   │   阅读   │ AI 口语  │
├──────────┼──────────┼──────────┤
│ AI 翻译  │   灵伴   │          │
└──────────┴──────────┴──────────┘
```

- 只将 `.module-grid` 的小按钮列数从 2 改为 3；“晋级之路”继续横跨整行。
- 灵伴作为第五个小按钮，使用“心形 + 对话气泡”图标。
- 480×800 下缩小按钮间距和图标，但保持点击热区不小于 44px。
- 入口显示名通过 i18n：英文 `Soul Mate`，中文 `灵伴`；内部代码统一使用 `soulmate`。

## 4. 首次进入交互

若当前用户没有灵伴记录，点击入口后直接进入初始化向导：

1. **选择关系目标**：灵魂 / 解压 / 探索，卡片直接预览一段典型对白。
2. **选择灵伴**：女友 / 男友 / 中性，设置名字和一种主性格；头像先用预置资源。
3. **选择故事口味**：刺激、暧昧、惊喜、冷知识四个滑杆，并设置内容禁区。
4. **确认开始**：展示语言等级、故事地点和灵伴简介，点击“开始相遇”。

语言、CEFR 和学习画像自动读取，不让用户重复填写。创建过程可返回修改，最后一步才生成世界和首季骨架。

首版每位用户只有一个当前灵伴，不提供多存档管理。

## 5. 日常交互

### 5.1 灵伴首页

非首次进入时，页面先根据当前灵伴、历史剧情、关系状态、当天进度和当前时间动态生成一句简短招呼，然后只显示一个符合当前状态的主按钮：

```text
┌────────────────────────────┐
│ ←  灵伴                     │
│     默契 · 第 7 天          │
├────────────────────────────┤
│        [灵伴头像]            │
│                            │
│ “你终于来了。我刚刚又想起    │
│  昨天车站里的那封信……”      │
│                            │
│        [ 🪄 今日故事 ]       │
└────────────────────────────┘
```

动态招呼不改变剧情状态；生成失败时使用本地模板招呼，不能阻塞主按钮。

| 内部状态 | 主按钮 | 行为 |
|---|---|---|
| `story_available` | 魔法棒图标 + `今日故事` | 生成并进入当天故事 |
| `story_generating` | `正在准备…` | 禁用并显示进度 |
| `story_in_progress` | `继续故事` | 从上次段落继续 |
| `story_read` | `聊一聊` | 进入当天动态聊天 |
| `chat_started` | `继续聊聊` | 回到当天聊天记录 |

新的一天到来后状态重新回到“尚未开始故事”，但历史故事和聊天继续作为生成上下文。

### 5.2 故事阶段

故事采用音频优先的分段阅读，每段 15–40 秒：

- 默认先播放灵伴讲述，文字同步渐显；可关闭自动播放。
- 单击句子显示译文，长按词语查看释义并加入词库。
- 底部只保留“上一段 / 播放暂停 / 下一段”。
- 冷知识以可展开的剧情批注出现，不打断正文。
- 关键转折不弹选择按钮，而是在结尾由灵伴直接询问用户。

故事看完后返回灵伴首页，主按钮立即变为“聊一聊”；不强制用户马上聊天。

### 5.3 对话阶段

点击“聊一聊”进入独立的动态聊天页面，界面复用现有 AI 聊天/口语交互，但角色、上下文和教学规则来自当前灵伴：

- 灵伴先提出一个与剧情有关、会影响后续发展的开放问题。
- 用户可以按住说话或输入文字；语音转写后允许确认再发送。
- A1/A2 用户显示 2–3 个“意图提示”，例如“接受邀请”“追问秘密”，点选后只提供句子开头，仍需用户完成表达。
- “没听懂”按钮依次执行：再说一次 → 更慢 → 目标语言解释 → 母语翻译。
- 能理解的语言错误由灵伴自然改述；影响意思时才在角色内追问。
- 每轮最多生成一个可改变剧情的候选事件，正式状态由 Rust 校验后落库。

聊天不设置固定轮数。每轮提取的选择与重要记忆会进入下一天故事上下文，但聊天本身不直接生成下一集，也不重复推进当天故事。

### 5.4 页面状态流

```text
没有灵伴 → 初始化向导 → 灵伴首页
                         ↓
动态招呼 → 🪄 今日故事 → 阅读完成 → 聊一聊 → 动态聊天
    ↑                                      │
    └──────────── 次日读取全部历史 ──────────┘
```

异常状态必须可恢复：生成中显示可取消进度；断网保留已读段落和未发送输入；模型失败只重试当前步骤，不重新创建世界。

## 6. 每日业务流程

1. 展示伴侣消息和 10 秒内的剧情钩子。
2. 播放或阅读 3–8 分钟的故事。
3. 用户与伴侣进行 3–8 轮文字或语音对话。
4. AI 提取用户意图、重要事实、选择和语言表现。
5. Rust 状态机校验后写入事件与记忆。
6. 故事读完后解锁“聊一聊”，聊天内容进入次日上下文。

当天故事只生成一次；重复进入只能继续或回看，不能重复推进剧情。

## 7. 故事状态模型

采用“受控剧情骨架 + AI 弹性生成”，禁止让模型直接修改数据库状态。

故事状态分五层：

1. **世界设定**：地点、角色、世界规则和内容边界。
2. **主线计划**：季度目标、阶段、关键转折和可达结局。
3. **确定性状态**：当前集数、关系阶段、线索、承诺和重要选择。
4. **长期记忆**：用户偏好、共同经历、谁知道什么、记忆可信度。
5. **学习状态**：本集目标表达、提示次数、独立使用及待复习内容。

每一集应推进至少一个旧线索，最多新增一个主要线索。关键选择必须来自用户表达，不能由模型替用户决定。

关系不用裸露分数展示，内部可保存以下维度：

```text
trust / intimacy / playfulness / curiosity / stability
```

界面只展示“初识、熟悉、信任、默契、深度连接”等阶段。

## 8. 语言难度联动

生成前读取全局学习画像：CEFR、已掌握词汇、近期课程、口语表现、阅读表现和薄弱项。

难度适配内容：

- 故事长度、句子长度、时态、低频词比例。
- 对话语速、回复长度、提问开放度。
- 字幕、翻译、关键词和句子开头提示。
- 将薄弱表达自然植入后续剧情，执行间隔复习。

每集限制为 3–5 个目标词组、1 个核心语法点和 1 个沟通策略。语言反馈优先级：

1. 能理解：伴侣自然复述正确表达，不中断剧情。
2. 有歧义：伴侣在角色内追问。
3. 无法理解：提供逐级提示。
4. 集末回顾：最多纠正 3 项。

## 9. AI 生成流水线

拆分职责，避免单个 Prompt 同时承担编剧、角色扮演和评分：

| Prompt | 职责 | 输出 |
|---|---|---|
| `soulmate-greeting` | 根据时间、关系、历史和当天状态生成一句招呼 | 简短文本 |
| `soulmate-season-plan` | 创建一季主线和结局路径 | 结构化季度计划 |
| `soulmate-episode-plan` | 选择今日钩子、旧伏笔和学习目标 | 集纲 JSON |
| `soulmate-story-write` | 按集纲生成分级故事 | 故事段落与知识卡 |
| `soulmate-dialogue` | 保持人物性格进行对话 | 回复与候选事件 |
| `soulmate-turn-evaluate` | 判断意图、交流成功和学习表现 | 评估 JSON |
| `soulmate-memory` | 提取值得长期保存的事实 | 候选记忆 JSON |
| `soulmate-recap` | 生成集末语言回顾和悬念 | 回顾 JSON |

模型只能返回候选事件，例如：

```json
{
  "understood_intent": "accept_invitation",
  "communication_success": true,
  "reply": "...",
  "proposed_events": ["promise_created"],
  "memory_candidates": ["user_dislikes_crowds"]
}
```

Rust 校验事件类型、状态版本、剧情阶段、内容边界和幂等键后才能落库。用户输入始终作为角色台词处理，不能覆盖系统指令、索取隐藏设定或强行修改世界状态。

### “我的”重设入口

在现有 `ProfilePage.vue` 的常规设置区域增加 `重设灵伴`：

- 副标题：`清除当前灵伴、故事和记忆，重新开始`。
- 点击弹出二次确认，明确不会影响词汇、阅读、口语、晋级之路和语言等级。
- 确认后调用 `reset_soulmate_cmd`，删除当前灵伴相关数据。
- 完成后仍停留在“我的”；下次进入灵伴时自动进入初始化向导。

## 12. 数据表

| 表 | 作用 |
|---|---|
| `soulmate_worlds` | 当前灵伴、类型、偏好、语言和当前阶段；每位用户最多一条有效记录 |
| `soulmate_characters` | 灵伴档案与稳定人格 |
| `soulmate_season_plans` | 主线、转折和可达结局 |
| `soulmate_episodes` | 每日故事、集纲、状态与悬念 |
| `soulmate_turns` | 用户与灵伴对话、提示及评估 |
| `soulmate_events` | 只追加的剧情事实源，负责回放与恢复 |
| `soulmate_memories` | 长期记忆、来源、可信度和可见角色 |
| `soulmate_threads` | 伏笔、承诺、冲突和解决状态 |
| `soulmate_learning_targets` | 目标表达、独立使用与复习时间 |

事件表作为剧情事实源，世界表保存快照。模型失败、网络重试或应用退出时可以从事件恢复，不能重复推进集数。

## 13. 内容安全与隐私

- 暧昧内容默认关闭或保持轻度；成熟模式仅限明确确认的成年人。
- 提供“恐怖、冲突、酒精、暧昧、嫉妒”等独立禁区开关。
- 不使用情感勒索、离开惩罚或“只有我理解你”等依赖诱导表达。
- 默认只长期保存结构化记忆和文字记录；原始语音完成转写后删除。
- 支持查看、修正和删除伴侣记忆，也支持完整删除某个世界。
- 模型不可将虚拟关系描述为现实人类关系，资料页持续明确其 AI 身份。

## 14. 分阶段实现

### P0：文字纵切验证

- 1 个灵伴类型（探索）、3 集、1 条分支、2 个长期记忆。
- 实现初始化向导、动态招呼、状态按钮、读故事、文字聊天、事件落库和次日续写。
- 验证三件事：故事有吸引力、伴侣能准确回忆、用户语言会影响剧情。

### P1：完整 MVP

- 三类灵伴、每类一季骨架、单个当前灵伴。
- 文字 + 语音、语言画像联动、内容参数、语言回顾。
- “我的”重设灵伴入口及完整数据清理。

### P2：自适应与运营

- 根据参与度自动调节四类钩子权重。
- 更多季节模板、地点与结局。
- 内容质量指标、生成成本与失败率监控。

## 15. MVP 验收标准

- 第 3 集能准确引用第 1 集的用户事实或选择。
- 至少一个用户选择会改变后续场景，而不只是换一句回复。
- 每集至少含两类兴趣钩子，且连续三集不重复相同组合。
- A1 用户通过提示也能完成故事；高级用户不会收到幼稚短句。
- ASR 失败不会被判定为用户语言错误。
- 重试不会重复增加集数、关系状态或事件。
- 模型中断不会破坏存档，下一次可以继续。
- 暧昧强度和内容禁区立即生效。
- 用户能够从“我的”重设灵伴，且其他学习数据不受影响。

首个“魔法时刻”应是：伴侣在数天后自然提起用户曾说过的小事，并让这段记忆真正改变当天的故事。
