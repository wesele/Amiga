🧠 内容生产体系

功能与交互只是外壳，学习内容才是核心资产。Idioma 采用语料库 + AI 编排策略：从开源语料库提取原始素材，经 AI 清洗、分级、编排、生成练习，最终存入数据库。核心内容预先生成，个性化练习实时生成，兼顾质量与灵活性。

📡

语料库驱动

从 Tatoeba、CC-CEDICT、Wiktionary、Leipzig Corpora 等开源语料库获取原始双语例句、词汇表和语音数据，确保内容来源合法合规。

🤖

AI 编排

LLM 自动清洗去重、CEFR 分级、提取核心词汇、编排课程单元、生成 10 种练习题型。人工审核抽检保证质量。

⚡

混合生成

核心课程内容预生成并缓存到本地数据库（约 80%），个性化错题复习、AI 口语对话等按需实时生成（约 20%），兼顾速度与个性化。

🔓

开放许可

所有使用的语料库均为 CC 或类似开放许可，符合商业和非商业使用要求。生成内容版权归平台与用户共有，支持社区贡献回源。

1. 语料采集

按语言对从各语料库拉取原始数据，统一为 SentencePair 格式（源语言句 + 目标语言句 + 来源标记 + 元数据）。支持增量更新和全量重建两种模式。

🧹

2. 清洗去重

LLM 自动过滤低质量句（过短/含乱码/特殊字符）、模糊去重、标准化标点格式。保留置信度分数供后续参考，低于阈值的进入人工审核队列。

📊

3. CEFR 分级

对每句话标注 A1~C1 级别，同时提取核心词汇、语法点。LLM 根据词汇复杂度、句子长度、语法结构综合判断。每个级别标注附带置信度。

📋

4. 课程编排

将同级别的素材按主题分组（问候、购物、旅行……），按难度递进排序，生成 Course → Unit → Skill 三层结构。每个 Skill 包含 5~10 句 + 3~8 个词汇。

✍️

5. 练习生成

基于素材自动生成 10 种练习题型：中译英、英译中、选词填空、拖拽排序、听力选择、语音跟读、配对连线、拼写纠错、阅读理解、造句。每课至少 8 道练习。

✅

6. 质量审核

自动规则校验（答案正确性、选项合理性、难度匹配）+ 人工抽检（每批次 5%）。抽检不合格的批次标记为草稿，返回管线重新生成。

⚡ 混合生成策略

🔋 预生成（离线）

占全部内容的 ~80%。在首次安装或版本更新时，通过后台管线批量生成所有核心课程、标准练习、词汇表。结果存入本地 SQLite 数据库，用户学习时零延迟读取。

✅ 覆盖：晋级之路全部课程、标准练习、词汇表、语法说明

⚡ 实时生成（在线）

占全部内容的 ~20%。由 LLM 在用户使用过程中按需生成，针对用户的薄弱环节和上下文做个性化定制。

✅ 覆盖：错题扩展练习、AI 口语对话、个性化复习、造句评测

📌 示例：A1 第 1 课 "打招呼"

以下是一个完整的从语料采集到入库的真实路径示例：

阶段输入 → 输出
语料采集Tatoeba: "Hello, how are you?" ↔ "你好吗？"
清洗去重→ "Hello, how are you?" / "你好吗？" ✅ 通过
CEFR 分级→ A1 (词汇: hello/how/are/you, 语法: 问候句式)
课程编排→ Unit: 基础问候 → Skill: 打招呼 (A1-01)
练习生成 中译英: "你好" → "Hello"
排序: "you / are / how / ?" → "How are you?"
配对: hello↔你好, goodbye↔再见, thank↔谢谢
听力: 听 "How are you?" → 选择中文翻译
质量审核✅ 答案校验通过 | ⭐ 难度匹配 A1 | ✅ 人工抽检状态: 待检
入库→ exercises 表存入 8 道练习, vocab 表存入 5 个词汇

✅ 质量保障体系

🤖

自动校验规则

每个生成结果经过多重验证：答案唯一性检查、选项不重复、错误选项合理性（与正确项属于同类但语义不同）、句子完整性检查。通过率低于 60% 的批次整批重跑。

👤

人工抽检流程

按批次抽检 5% 样本，检查：翻译准确性、CEFR 级别匹配度、练习趣味性、文化适当性。不合格批次标记为 draft 并返回重新生成。

📊

用户反馈闭环

每道练习附带"反馈"按钮（内容有误/太难/太简单/翻译不对），用户反馈自动汇总，触发该题/该课的重新审核和再生成。

🔄

版本迭代

语料源更新或 AI 模型升级后，标记受影响课程为"stale"，在后台按优先级重新编排和生成，通过审核后静默替换用户本地缓存。

🏗️ 技术架构

🖥️

后台管线 (Tauri Rust)

语料采集、清洗、分级、编排、生成均在 Tauri 后台 Rust 进程执行。支持异步并行处理，利用 SQLite 作为中间存储和最终存储。AI 调用通过 HTTP 请求 LLM API。

📱

前端运行时 (Vue 3)

用户学习时从本地 SQLite 读取预生成内容，零网络延迟。需要实时生成时（AI 口语/错题复习），通过 Tauri invoke 调用 Rust 命令，Rust 再请求 LLM API。

🧠

LLM 接口层

统一 Prompt 模板和 Schema 定义，支持切换不同 LLM 后端（GPT-4、Claude、本地模型）。每条生成请求附带结构化输出约束，便于直接解析入库。

🗄️

数据库

SQLite 存储预生成内容。表结构复用 courses、units、skills、exercises、vocabulary 等现有设计。新增 pipeline_jobs、corpus_sources、review_log 表管理管线流程。
